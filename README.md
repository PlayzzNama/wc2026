# WC2026 Predictor — Міні-додаток для прогнозів на Чемпіонат світу 2026

Міні-додаток для Telegram, в якому ви з братом (Іван і Роман) можете прогнозувати результати всіх матчів ЧС-2026. Додаток працює як всередині Telegram, так і у звичайному браузері.

## Правила нарахування балів

- **+5 балів** — якщо вгадав **точний рахунок** матчу
- **+2 бали** — якщо правильно вгадав **переможця** або нічию (але рахунок не точний)
- **-2 бали** — якщо не вгадав ні переможця, ні нічию

## Початкові бали (засіяні при першому запуску)

- **Іван**: 2 бали  
  (вгадав переможця у перших двох матчах по +2, і -2 за третій матч)
- **Роман**: 8 балів  
  (вгадав точний рахунок у перших двох матчах по +5, і -2 за третій матч)

Наразі в додатку додано **71 матч** групового етапу за наданим тобою розкладом (11–27 червня 2026).

## Встановлення та запуск (локально)

### 1. Перший запуск (рекомендовано)

```powershell
# Перейди в папку проєкту
cd D:\Frontend\WCGame

# Скопіюй приклад файлу налаштувань (вже містить твій токен бота)
Copy-Item .env.example .env

# (Важливо!) Відкрий файл .env і вкажи реальні ID користувачів
# ADMIN_IDS=твій_telegram_id,roman_telegram_id
```

### 2. Запусти проєкт

```powershell
npm run dev
```

Ця команда запускає одночасно:
- Фронтенд (http://localhost:5173)
- Бекенд + Telegram бота (http://localhost:3001)

### 3. Як увійти в додаток

- Відкрий браузер і зайди на http://localhost:5173 (для тестування)
- Або в Telegram знайди свого бота і напиши команду **/start**

## Як користуватися додатком

1. Відкрий міні-додаток через бота (або локально).
2. У верхній частині екрана є перемикач **Іван / Роман** — можна переглядати прогнози та бали один одного.
3. У вкладці **Upcoming** — майбутні матчі. Вводь прогноз (рахунок господарів і гостей) і натискай **Save Prediction**.
4. У вкладці **Finished** — завершені матчі з реальним рахунком та нарахованими балами.
5. У вкладці **Leaderboard** — таблиця лідерів з поточними балами.
6. Якщо ти Іван (або твій ID вказано в `ADMIN_IDS`) — з’являється вкладка **Admin**.

> **Важливо**: коли ти відкриваєш додаток всередині Telegram — він автоматично визначає тебе за реальним Telegram ID. Прогнози завжди зберігаються під твоїм реальним ID.

## Адмін-панель (тільки для Івана)

У вкладці **Admin** доступні такі функції:

- **Set / Edit Result** — поставити або виправити результат матчу. Після цього матч стає завершеним і всім учасникам нараховуються бали.
- **Add New Match** — додати новий матч (дуже зручно для плей-офф).
- **Export & Copy JSON** — експортувати всі дані (на випадок резервної копії).
- **Import from Brother** — імпортувати дані, які надіслав брат.
- **Reset to Initial** — повернути все до початкового стану (засіяні бали).

---

## Як додавати нові матчі (плей-офф тощо)

1. Увійди як адміністратор.
2. Перейди у вкладку **Admin**.
3. Натисни **Add New Match**.
4. Заповни:
   - Home Team (господар)
   - Away Team (гість)
   - Date (дата та час у зручному форматі)
   - Stage (наприклад: "Round of 32", "Quarter-final", "Semi-final" тощо)
5. Натисни **Add Match**.

Пізніше, коли матч закінчиться — натисни біля нього **Set Result & Finish** і введи реальний рахунок.

Навіть якщо ти залогінився під своїм реальним ID, ти можеш натиснути на ім’я брата у верхньому перемикачі і подивитися:
- Які прогнози він поставив

- Що він вгадав у завершених матчах

При цьому твої власні прогнози завжди зберігаються під твоїм ID.

## Розгортання на продакшен (важливо!)

**Netlify / Vercel / GitHub Pages** — це тільки статичний хостинг для фронтенду. Вони **не вміють** запускати Node.js бекенд (`server/server.js` з Express і ботом).

Тому треба розгортати **фронтенд** і **бекенд окремо**.

### Крок 1: Розгорнути бекенд (обов’язково)

Рекомендовані безкоштовні сервіси для бекенду:

- **Render.com** (найпростіший)
- **Railway.app**
- **Fly.io**

**Як розгорнути на Render (рекомендую):**

1. Зайди на [render.com](https://render.com), увійди через GitHub.
2. Натисни **New + → Web Service**.
3. Вибери свій репозиторій.
4. Налаштування:
   - **Name**: `wc2026-backend`
   - **Runtime**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `node server/server.js`
   - **Plan**: Free
5. Додай Environment Variables:
   - `BOT_TOKEN` = `8977354914:AAFrmrKNnZC9fxnRDyvtIlPvj6-82uOKsPg`
   - `ADMIN_IDS` = `948806223`   (тільки твій ID)
   - `WEBAPP_URL` = `https://earnest-cendol-5c3bf1.netlify.app`  (адреса твого фронтенду)
   - `PORT` = `10000` (Render сам підставляє)

6. Натисни Deploy.

Після деплою скопіюй URL бекенду (наприклад `https://wc2026-backend.onrender.com`).

### Крок 2: Налаштувати фронтенд на Netlify

1. Зайди в налаштування свого сайту на Netlify.
2. **Site configuration → Environment variables**:
   - Додай змінну:
     - **Key**: `VITE_API_URL`
     - **Value**: `https://wc2026-backend.onrender.com`  (URL твого бекенду без слеша в кінці)
3. Натисни **Deploy site** або зроби новий деплой (`git push`).

### Крок 3: Оновити WEBAPP_URL у бота

- Перейди в Render → Environment variables бекенду.
- Постав `WEBAPP_URL` = `https://earnest-cendol-5c3bf1.netlify.app`
- Передеплой бекенд (або просто перезапусти сервіс).

### Перевірка

- Відкрий свій додаток через Telegram.
- Спробуй поставити прогноз.
- У консолі браузера (F12) дивись Network — запити повинні йти на твій бекенд, а не на netlify.

Якщо все ще будуть помилки — надішли скріншоти з консолі браузера (вкладка Network + Console) і з логів бекенду на Render.

## Вирішення поширених проблем

**Бот не надсилає кнопку /start**
- Перевір, чи правильно вказаний `WEBAPP_URL` у `.env` (повинен бути HTTPS).
- Перезапусти сервер після зміни `.env`.

**Не бачу вкладку Admin**
- Твій Telegram ID не додано в `ADMIN_IDS`.
- Або використовуєш демо-режим (ID 1 і 2 працюють як адміни автоматично).

**Дані не синхронізуються між пристроями**
- Переконайся, що обидва користуєтеся однією адресою продакшену.
- При локальній розробці дані зберігаються у файлі `data/db.json`.

**Хочу скинути всі дані**
- Зупини сервер.
- Видали файл `data/db.json`.
- Запусти сервер знову — він засіє початкові дані.
